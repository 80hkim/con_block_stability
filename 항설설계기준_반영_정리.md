# 항설 설계기준 반영 정리

## 1. 반영 방향

이번 수정은 항만분야 검토기의 역할을 다음과 같이 정리하는 방향으로 수행하였다.

- 활동, 전도, 직선활동은 항만 및 어항 설계기준 기준값으로 검토
- 지지력은 `종래의 지지력 산정식 적용`에 따라 `하중분산법 2단계`로 검토
- 원호활동 검토는 이 앱에서 직접 계산하지 않고, 지반분야 별도 프로그램 수행 전제로 명시

## 3. 주요 수정 사항

### 3.1 활동, 전도, 직선활동 지진시 기준 수정

- 기존: 지진시 `Fs ≥ 1.0`
- 수정: 지진시 `Fs ≥ 1.1`

반영 파일
- `js/stability.js`
- `index.html`
- `js/main.js`
- `js/ai_chat.js`

### 3.2 지지력 검토 방식을 종래식으로 변경

기존 지지력 검토는 사실상 `qmax ≤ qa`와 편심 조건 중심의 단순 판정이었다.

이번에는 종래식 하중분산법 2단계 구조로 정리하였다.

- 1단계: 사석 마운드 상면에서 `p1 ≤ qta`
- 2단계: 기초지반 상면에서 `p1' ≤ qa`
- 최종 판정: 위 두 조건을 모두 만족할 때만 `O.K`

즉 결과표의 지지력 항목은 이제 다음 의미를 가진다.

- `p1`: 사석 마운드 상면 최대접지압
- `p1'`: 기초지반 상면 최대응력
- `qta`: 사석 마운드 상면 허용지지력
- `qa`: 기초지반 상면 허용지지력

반영 파일
- `js/stability.js`
- `js/main.js`
- `js/ai_chat.js`
- `index.html`

### 3.3 지지력 입력값 분리

기존에는 `qa`만 입력했다.

이번에는 다음 입력으로 분리하였다.

- 상시 / 지진시 사석 마운드 상면 허용지지력 `qta`
- 상시 / 지진시 기초지반 상면 허용지지력 `qa`

반영 파일
- `index.html`
- `js/main.js`

### 3.4 결과표와 툴팁 수정

결과표 지지력 셀은 기존 `qmax` 단일값에서 다음 형식으로 변경하였다.

- `p1 / p1'`

기준 셀은 다음 형식으로 표시되도록 변경하였다.

- `qta / qa`

툴팁에는 아래 정보를 표시한다.

- `p1`, `qta`
- `p1'`, `qa`
- 편심 `e`
- 반력분포 폭 `b`
- 하중 확산폭 `L`
- 원호활동은 별도 프로그램 수행이라는 안내

반영 파일
- `index.html`
- `js/main.js`

### 3.4-1 가상배면 해측 뒤채움사석 하중 반영 수정

기존에는 가상배면 내부의 해측 토량을 사실상 `매립토` 자중으로 취급하고 있었다.

이번에는 항설 취지에 맞춰 다음과 같이 수정하였다.

- 가상배면 기준 해측에 존재하는 토량은 `뒤채움사석`으로 본다
- 이 하중은 검토 단면마다 상부에 남는 부분만 분할하여 수직하중으로 반영한다
- 지진시에는 같은 구간을 뒤채움사석 관성력으로도 반영한다

즉 현재 엔진은 가상배면 해측 토량을 `매립토 자중`이 아니라 `뒤채움사석 자중`으로 취급한다.

반영 파일
- `js/stability.js`
- `js/main.js`

### 3.5 최적화 비교표의 지지력 표시 수정

최적화 결과표도 종래식 지지력 형식에 맞게 변경하였다.

- 상시: `p1 / qta`, `p1' / qa`
- 지진시: `p1 / qta`, `p1' / qa`

지지력 O.K/N.G 판정은 실제 계산 엔진의 `bearing.pass`를 그대로 사용한다.

반영 파일
- `js/ai_chat.js`

### 3.6 AI 설명 규칙 수정

AI가 기준 밖 설명을 하지 않도록 다음 규칙을 명시하였다.

- 하중조합: `상시 비계류` / `상시 계류(견인력 시)` / `지진시` (8 또는 12조합)
- 지지력은 `p1`, `p1'`, `qta`, `qa` 기준으로만 설명
- `FS_br = qa/qmax` 같은 임의 식 생성 금지
- 원호활동은 이 앱에서 직접 계산하지 않고 지반분야 별도 검토 항목으로 설명

반영 파일
- `js/main.js`
- `js/ai_chat.js`

## 4. 현재 앱의 역할 정리

이 앱은 현재 다음 역할을 수행한다.

- 항만분야 안정검토기
- 종래식 지지력 산정
- 원호활동 별도 검토 전제의 결과 정리
- 지반분야 원호활동 수행팀에 넘길 기초 지지력 결과 정리

이 앱은 현재 다음은 수행하지 않는다.

- 원호 중심/반경 탐색
- 절편분할법 / 비숍법 직접 계산
- 원호활동 최소 안전율 판정

## 6. 확인 포인트

- 결과표의 지진시 활동/전도/직선활동 기준이 `≥1.1`인지
- 지지력 입력부에 `qta`, `qa`가 모두 분리되어 있는지
- 결과표 지지력 셀이 `p1 / p1'` 형식으로 표시되는지
- 툴팁에 `p1`, `qta`, `p1'`, `qa`, `b`, `L`가 표시되는지
- 화면 하단에 `원호활동은 별도 프로그램 수행` 안내가 있는지
- NG 원인분석에서 `FS_br`, `qa/qmax`, `지지력 안전율 2.0` 같은 문구가 나오지 않는지
